„Insights" — popover czy okno? Oba kierunki, równej rangi.

Model nawigacji przesądzony (ścieżka Dockera): menu systemowe to pilot, Insights to dom — wchodzi się weń z menu albo z powiadomienia, nie klikiem w ikonę. Zostało jedno pytanie, i ten dokument ma je rozstrzygnąć na kadrach: czym ma być powierzchnia Insights.

Węzeł = notatka
Łuk = połączenie
Rozbłysk = insight
A

Popover

„zerknąć i zniknąć" — karta pod ikoną

  • ×Klikniesz obok → zamyka się
  • ▣Stały, mały rozmiar (~340 px)
  • 1Jedno połączenie naraz, nawigacja ‹ N ›
  • ↑Build niski — mamy NSPopover

Wychodzisz do Obsidiana → popover gaśnie. Czytasz z pamięci.

B

Okno (Dashboard)

„usiąść, czytać, wracać" — dom dla całego korpusu

  • ✓Zostaje aż je zamkniesz; drugi monitor, wiele naraz
  • ⤢Resizable, duże; lista N połączeń obok czytnika
  • ∞Przeglądasz korpus, nie jedno połączenie
  • ↑Build większy — NSWindow + chrome

Wychodzisz do Obsidiana → okno trwa obok edytora.

Argument rozstrzygający: chipy notatek linkują do Obsidiana — użytkownik wychodzi z apki w trakcie czytania. Sekcje A i B pokazują tę samą treść i tę samą konstelację; różnią się tylko zachowaniem. Klatki „konsekwencji" (§A·4, §B·3) czynią różnicę widzialną.

S

Systemowe menu — pilot (natywne, nie projektujemy)

Support · chrome systemu

Klik w ikonę menu-bar otwiera natywne NSMenu (jak Docker / Cursor). Status-header + dwa wejścia do Insights. Rysujemy je tylko wiernie — system renderuje je sam.

A

Insights jako popover

Kierunek do decyzji

NSPopover spod ikony. Jedno połączenie naraz, ‹ 1/N ›. Trzy typy — różny układ węzłów, ta sama gramatyka.

A·stany

Walidacja, cisza i konsekwencja

Zachowaj = mikro-rozbłysk i „zachowane". Po przejrzeniu — spokojny pusty. Klik w chip notatki → popover gaśnie (to jego ograniczenie).

B

Insights jako okno — Dashboard

Kierunek do decyzji

To dom. Lewa szyna = lista połączeń (przeglądasz korpus); czytnik = żywa konstelacja + rationale; dół = ostatnie transkrypty. Kadr po rozbłysku, N=7.

B·stany

Pusto i konsekwencja

Pusty stan: spokojny, nie smutny. Konsekwencja: okno trwa obok otwartego Obsidiana — czytasz jednocześnie.

sygnał

Menu-bar i powiadomienie

Support · wspólne dla A i B

Ikona z dyskretną złotą poświatą, gdy czeka niezobaczony insight. Powiadomienie niesie samą tezę — nie „New digest ready". Klik → otwiera Insights.

Ikona menu-bar · dwa stany
Spoczynek — nic nie czeka
Czeka insight — złoty punkt + poświata
Powiadomienie · niesie tezę, nie etykietę · klik → otwiera Insights
